the safest way to access an attribute is prefix that attribute's name with an underscore. this helps prevent clashes with any class methods with the same name. you can also get directly to the attribute without using the underscore, but you should be familiar with pymel because if you try to access an attribute that has the same as a class method of your node, you will raise an error. for example: 
>>> sphere = polySphere()[0]
>>> sphere.translate.set(1,2,3) # this conflicts with a method inherited from the base string class
AttributeError: 'function' object has no attribute 'set'
>>> sphere._translate.set(1,2,3) # this succeeds

this happens because rotate is a method of the sphere's transform class. use this feature at your own risk!

modifying attributes follows a fairly simple pattern. 'setAttr' becomes 'set', 'getAttr' becomes 'get', 'connectAttr' becomes 'connect' and so on. there are also handy operators for connect, disconnect, and isConnected. 
>>> sphere = polySphere()[0]
>>> cube = polyCube()[0]                
>>> sphere.tx >> cube.tx        # connect
>>> if sphere.tx >>= cube.tx:
...     print "is connected!"
is connected!
>>> sphere.tx <> cube.tx        # disconnect

insertInput(self, node, nodeOutAttr, nodeInAttr)

source code  
connect the passed node.outAttr to this attribute and reconnect
any pre-existing connection into node.inAttr.  if there is no
pre-existing connection, this method works just like connectAttr. 

for example, for two nodes with the connection:
        a.out-->b.in
        
running this command:
        b.insertInput( 'c', 'out', 'in' )
        
causes the new connection order (assuming 'c' is a node with 'in' and 'out' attributes):                
        a.out-->c.in
        c.out-->b.in
