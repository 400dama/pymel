 11  Pymel 
 12  ===== 
 13   
 14  Pymel makes python scripting with Maya work the way it should. Maya's command module is a direct 
 15  translation of mel commands into python commands. The result is a very awkward and unpythonic syntax which 
 16  does not take advantage of python's strengths -- particulary, a flexible, object-oriented design. Pymel 
 17  builds on the cmds module by organizing many of its commands into a class hierarchy, and by 
 18  customizing them to operate in a more succinct and intuitive way. 
 19   
 20   
 21  Getting Started 
 22  =============== 
 23   
 24  If you are a mel scripter but have not used python in maya yet, you should start with the Maya docs on the subject, particularly 
 25  the section 'Using Python'. This will help you to understand the differences in syntax between the two languages and how  
 26  to translate between them. 
 27   
 28  The Basics 
 29  ---------- 
 30   
 31  In its current incarnation, pymel is designed with a great deal of backward compatibility in mind, so that the maya.cmds 
 32  module and the pymel module can be used interchangably with the same code.  However, a closer look reveals pymel is actually 
 33  hiding a great deal of its power right under your nose.  Take the L{ls} command for example.  L{maya.cmds.ls}  
 34  will return a list of strings.  These strings have a lot of built-in functionality that make them a million times more  
 35  powerful than strings in mel: 
 36   
 37          >>> import maya.cmds 
 38          >>> cam = maya.cmds.ls( type='camera')[0] 
 39          >>> print cam 
 40          frontShape 
 41          >>> cam.replace( 'front', 'crazy' )  # an example string operation 
 42          # Result: crazyShape #  
 43                   
 44  So, already you have object-oriented power at your fingertips. When using pymel, the L{ls} command still returns a list of strings, 
 45  but these strings are on steroids: in addition to the built-in string methods ( a method is a function that belongs to a class ),  
 46  pymel adds methods for operating on the type of object that the string represents: 
 47   
 48          >>> import pymel 
 49          >>> cam = pymel.ls( type='camera')[0] 
 50          >>> print cam 
 51          frontShape 
 52          >>> cam.getFocalLength() 
 53          # Result: 35.0 #  
 54          >>> cam.replace( 'front', 'crazy' )  # still has the string functionality as well 
 55          # Result: monkeyShape # 
 56   
 57  The same goes for other objects when using pymel.  When getting a triple attribute like translate or rotate, maya.cmds.getAttr 
 58  will return a list with three floats.  Pymel nodes, on the other hand, return a 3-element L{vector<MVec>} which inherits from the list class, 
 59  so that it has all the power of a list, and is backward compatible with any functions that expect to receive lists, but which 
 60  also has the ability to do vector and matrix math functions.  
 61   
 62   
 63  Object-Oriented Design 
 64  ====================== 
 65   
 66  The pymel module reorganizes many of the most commonly used mel commands into a hierarchy of classes. This design allows  
 67  you to write much more concise and readable python code. It also helps keep all of the commands organized, so that 
 68  functions are paired only with the types of objects that can use them. 
 69   
 70  All node classes inherit from the L{Node} class.  
 71           
 72  Understanding the L{Attribute} class is essential to using pymel to its fullest extent. 
 73           
 74  Using Node Classes 
 75  ------------------ 
 76   
 77          In order to use the object-oriented design of pymel, you must ensure that the objects that you are working  
 78          with are instances of pymel classes. To make this easier, this module contains wrapped version  
 79          of the more common commands for creating and getting lists of objects which cast their results to the appropriate  
 80          class type. See L{ls}, L{listRelatives}, L{listTransforms}, L{selected}, and L{listHistory}.   
 81           
 82                  >>> s = ls(type='transform')[0] 
 83                  >>> print type(t) 
 84                  <class 'pymel.core.Transform'> # 
 85                   
 86          Most commands that create objects are wrapped as well (see below): 
 87           
 88                  >>> t = polySphere()[0] 
 89                  >>> print t, type(t) 
 90                  pSphere2, <class 'pymel.core.Transform'> # 
 91                   
 92          In many cases, you won't be creating the object directly in your code, but will want to gain access to the object by name. Pymel 
 93          provides two new ways of doing this. 
 94                   
 95          Using Objects by Name: The PyNode Command 
 96          ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ 
 97           
 98          The L{PyNode} command will automatically choose the correct pymel class for your object. 
 99           
100                  >>> s = PyNode('perspShape') # convert to a pymel class 
101                  >>> print s, type(s) 
102                  perspShape, <class 'pymel.Camera'> #  
103   
104   
105   
106          Using Objects by Name: The SCENE object 
107          ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ 
108                   
109          The 'SCENE' object provides the same functionality as the PyNode command, but with a slightly different syntax. 
110   
111          with PyNode: 
112                  >>> PyNode( 'defaultRenderGlobals').startFrame.get() 
113                  # Result: 1.0 # 
114   
115          with SCENE: 
116                  >>> SCENE.defaultRenderGlobals.startFrame.get() 
117                  # Result: 1.0 #

121  Object Commands and their Classes 
122  --------------------------------- 
123   
124  Mel contains a number of commands which are used to create, edit, and query specific object types in maya.   
125  Typically, the names of these commands correspond with the node type on which they operate.  some examples of these 
126  commands are aimConstraint, camera, directionalLight, and polySphere ( see commandsCreation in the pymel directory 
127  for a complete list). Pymel creates a class for each of these commands -- the class name is simply the command capitalized. 
128  Once nodes have been cast to their appropriate type (usually handled automatically) the commands operations can 
129  be performed in an object oriented way. 
130   
131  this example demonstrates some basic principles. note the relationship between the name of the object 
132  created, its node type, and its class type 
133   
134          >>> l = directionalLight() 
135          >>> # print the name of the object, its maya object type, and the class type 
136          >>> print l, l.type(), type(l)   
137          directionalLightShape1 directionalLight <class 'pymel.DirectionalLight'> 
138   
139          >>> # make the light red, the old school way 
140          >>> directionalLight( l, edit=1, rgb=[1,0,0] )  
141          >>> # make the light red, the pymel way 
142          >>> l.setRgb( [1,0,0] ) 
143   
144          >>> # get the shadow samples, the old school way 
145          >>> directionalLight( l, query=1, shadowSamples=1 )  
146          1 
147          >>> # get the number of shadow samples, the pymel way 
148          >>> print l.getShadowSamples()    
149          1 
150   
151  Immutability 
152  ------------ 
153   
154  All node classes are subclasses of python's built-in unicode string type, which allow them to be easily printed, passed to  
155  commands and used as keys in dictionaries. However, since strings are immutable, when calling  
156  commands like L{rename}, the calling instance will point to an invalid object after the rename, so you  
157  need to assign the result of the command to a new instance. For example: 
158   
159          >>> orig = polySphere()[0] 
160          >>> new = orig.rename('crazySphere') 
161          >>> print orig.exists(), new.exists() 
162          0 1 
163           
164  Delving Deeper: Chained Node Lookups 
165  ------------------------------------ 
166   
167  Mel provides the versatility of operating on a shape node via its transform node.  for example, these two commands work 
168  interchangably:: 
169   
170          camera -q -centerOfInterest persp 
171          camera -q -centerOfInterest perspShape 
172   
173  pymel achieves this effect by chaining function lookups.  If a called method does not exist on the Transform class, the  
174  request will be passed to appropriate class of the transform's shape node, if it exists. 
175  The chaining goes one further for object primitives, such as spheres, cones, etc.  For example: 
176           
177          >>> # create a sphere and return its transform 
178          >>> trans = polySphere()[0] 
179          >>> print type(trans) 
180          <class 'pymel.core.Transform'> 
181          >>> # get the transform's shape 
182          >>> shape = trans.getShape() 
183          >>> print type( shape ) 
184          <class 'pymel.core.Poly'> 
185          >>> # get the shape's history 
186          >>> hist = shape.history()[1] 
187          >>> type( hist ) 
188          <class 'pymel.PolySphere'> 
189          >>> # get the radius of the sphere  
190          >>> hist.getRadius() 
191          1.0  
192          >>> # chained lookup allows the PolySphere.getRadus method to work on the Transform class   
193          >>> trans.getRadius() 
194          1.0  
195   
196  the method getRadius belongs to the PolySphere class.  In this example, getRadius does not exist on the Transform class, so it passes 
197  the request to its shape, which is a Poly class. The method does not exist here either, so the Poly class searches for its primary 
198  construction history node, which is the polySphere node.  This node is cast to a PolySphere class which has the desired getRadius method, 
199  which is then called.

201  Mel Scripts 
202  =========== 
203  Calling mel scripts through maya.mel.eval is a nuisances because it requires so much string formatting on  
204  the programmer's part.  L{pymel.mel} handles all of that for you so you can use your mel scripts as if they  
205  were python functions. This includes automatically formatting all iterable types into maya arrays. see 
206  L{pymel.core.Mel} for more information. 
207   
208   
209  Module Namespaces 
210  ================= 
211   
212  Another problem with maya.cmds is that importing it into the root namespace (e.g. 'from maya.cmds import *') 
213  is dangerous because it will override several of python's more important built-in methods. pymel is designed 
214  to be safe to import into the root namespace so that scripts can be written much more concisely.  
215   
216  All the functions in maya.cmds are in the pymel namespace, except the conflicting ones ( file, filter, eval, 
217  help, and quit). The conflicting commands can be found in the pymel.cmds namespace, along with all of the unaltered 
218  maya commands.   
219   
220  Also, see L{pymel.io} for more information on how the file command is implemented in pymel. 
221   
222  Even though pymel has a handful of modules, they are all imported directly into the main namespace, except for the ctx module. 
223  In the future, I will try to establish a way for the user to easily customize which modules are directly imported and 
224  which should remain in their own namespace for organizational reasons. 
225

227  Setup 
228  ===== 
229   
230  if you have not done so already, it is a good idea to setup a directory for your published python scripts so that 
231  they will be accessible from within maya.  you do this by setting the PYTHONPATH environment variable in Maya.env.   
232  if you want to use your current maya scripts directory simply add this to your Maya.env:: 
233   
234          PYTHONPATH = $MAYA_SCRIPT_PATH 
235   
236  then place the pymel directory in your PYTHONPATH directory. 
237   
238  next, to avoid having to import pymel every time you startup, if you have not done so already, you can create a userSetup.mel 
239  file, place it in your scipts directory and add this line:: 
240   
241          python("from pymel import *"); 
242   
243  Note, that if you are running a linux/unix variant, including osx, and you have your PYTHONPATH set in a shell resource file,  
244  this value will override your Maya.env value if you launch maya from the terminal.
