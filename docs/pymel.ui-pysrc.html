  3  Pymel UIs 
  4  ========= 
  5   
  6  pymel adds more readability to ui building while also maintaining backward compatibility.  Every ui command in maya.cmds 
  7  is a class in pymel, which can behave like a command or like a class. More documentation on this to come, but for now 
  8  check out pipeGen.py in examples directory 
  9   
 10  Command Callbacks 
 11  ----------------- 
 12   
 13  one common point of confusion is command callbacks with ui elements. There are several different ways to handle  
 14  command callbacks on user interface widgets:   
 15                                                   
 16  Function Name as String 
 17  ~~~~~~~~~~~~~~~~~~~~~~~ 
 18   
 19  maya will try to execute this as a python command, but unless you know the namespace it will 
 20  be imported into, the function will not be recognized. notice how the namespace must be hardwired 
 21  into the command: 
 22   
 23          >>> button( c="myCommand" ) 
 24   
 25  or 
 26   
 27          >>> button( c="myModule.myCommand" ) 
 28   
 29  this method is not recommended. 
 30   
 31  Function Object 
 32  ~~~~~~~~~~~~~~~   
 33   
 34  When using this method, you pass an actual function object (without the parentheses). The callback function 
 35  has to be defined before it is referenced by the command flag.  also, keep in mind many ui widgets such as radioButtonGrp 
 36  pass args to the function (to mimic the "myCommand #1" functionality in mel), which your function must accommodate. The 
 37  tricky part is that different ui elements pass differing numbers of args to their callbacks, and some pass none at all. 
 38  This is why it is best for your command to use the *args syntax to accept any quantity of args, and then deal with them 
 39  in the function. 
 40   
 41          >>> def myCommand( *args ): print args # this definition must come first 
 42   
 43          >>> button( c=myCommand ) 
 44                                   
 45  Lambda Functions 
 46  ~~~~~~~~~~~~~~~~ 
 47  In my experience this is the best way to handle most command callbacks.  You can choose exactly which args you want 
 48  to pass along to your function and order of definition does not matter. 
 49   
 50          >>> button( c= lambda *args: myCommand(args[0]) ) 
 51   
 52          >>> def myCommand( arg ): print "running", arg
