Pymel

Pymel makes python scripting with Maya work the way it should. Maya's command module is a direct translation of mel commands into python commands. The result is a very awkward and unpythonic syntax which does not take advantage of python's strengths -- particulary, a flexible object-oriented design. Pymel builds on the maya.cmds module by organizing many of its commands into a class hierarchy, and by customizing them to operate in a more succinct and intuitive way.

Pymel Features:
  • powerful new classes for nodes, attributes, file paths, sets and vectors
  • customized operators for succinct scripting
  • allows mel procedures to be called as if they were python commands: no more annoying string formatting
  • mel-to-python translator for converting mel script into python scripts
  • python-to-mel plugin factory for turning python classes into mel commands
take a look at this simple comparison:

MEL

: 
       string $objs[] = `ls -type transform`;
       for ($x in $objs) {
               connectAttr( $x + ".sx") ($x + ".sy");
               disconnectAttr( $x + ".sx") ($x + ".sy");
               setAttr ($x + ".rotate") 1 1 1; 
               string $conn[] = `listConnections -s 0 -d 1 -p 1 ($x + ".tx")`;
               for ($inputPlug in $conn)
                       disconnectAttr $inputPlug ($x + ".tx");
               move 1 2 3 $x;
               float $trans[] = `getAttr ($x + ".translate")`;
               float $scale[] = `getAttr ($x + ".scale")`;
               $trans[0] *= $scale[0];
               $trans[1] *= $scale[1];
               $trans[2] *= $scale[2];
               setAttr ($x + ".scale") $trans[0] $trans[1] $trans[2];
               myMelScript( `nodeType $x`, $trans );
               print longNameOf($x);   
       }

Default Python

: 
       import maya.cmds as cmds
       import maya.mel as mm
       objs = cmds.ls( type= 'transform') 
       if objs is not None:                                                            # maya.cmds.ls returns None when it finds no matches
               for x in objs:
                       cmds.connectAttr( '%s.sx' % x,  '%s.sy' % x)
                       cmds.disconnectAttr( '%s.sx' % x,  '%s.sy' % x)
                       cmds.setAttr ( '%s.rotate' % x, 1,  1, 1 )
                       conn = listConnections( x + ".tx", s=0, d=1, p=1)
                       if conn:
                               for (inputPlug in conn)
                                       disconnectAttr( inputPlug, x + ".tx")   
                       cmds.move( 1, 2, 3, x )                 # confusing syntax
                       trans = cmds.getAttr ( '%s.translate' % x )
                       trans = trans[0]                                # maya packs the previous result list in a tuple for no apparent reason
                       scale = cmds.getAttr ( '%s.sclae' % x )[0]
                       trans[0] *= scale[0]
                       trans[1] *= scale[1]
                       trans[2] *= scale[2]
                       cmds.setAttr ( '%s.scale' % x, trans[0], trans[1], trans[2] )
                       mm.eval('myMelScript("%s",{%s,%s,%s})' % (cmds.nodeType(x), trans[0], trans[1], trans[2])
                       print mm.eval('longNameOf("%s")', x)

With Pymel

: 
       from pymel import *                                             # safe to import into main namespace
       for x in ls( type='transform'):
               x.sx >> s.sy                                            # connection operator
               x.sx <> s.sy                                            # disconnection operator
               x.tx.disconnect()                                       # smarter methods -- (automatically disconnects all inputs and outputs when no arg is passed)
               x._rotate.set( 1, 1, 1 )        
               x.move( 1, 2, 3 )
               trans = x._translate.get()
               trans *= x._scale.get()                         # vector math
               x._translate.set(trans)                         # ability to pass list/vector args
               mel.myMelScript( x.type(), trans)       # automatic handling of mel commands
               print x.longName()                                      # object oriented design
As you can see, pymel has a lot of little touches that make writing python code in maya a lot simpler -- as it should be. pymel handles automatic casting of arguments and return values as much as it can, so that you don't have to. for example, the 'get' method returns a simple vector type when returning triples from transforms so you can perform vector math on the fly. also, you can set triple values directly as a list, instead of having to unpack it (see the scale setAttr example above). Another nice feature is that pymel commands that return lists (such as ls and listRelatives) return empty lists instead of nothing when they find no match. that is why, in the above examples, the default python example must first place the 'ls' results in a variable and check if it is 'None', whereas the pymel example will simply not enter the for loop if there are no transforms in the scene.

Mel Scripts

calling mel scripts through maya.mel.eval is a nuisances because it requires so much string formatting on the programmer's part. pymel.mel handles all of that for you so you can use your mel scripts as if they were python functions. this includes automatically formatting all iterable types into maya arrays.

Module Namespaces

another problem is that importing maya.cmds into the root namespace (e.g. 'from maya.cmds import *') is dangerous because it will override several of python's more important built-in methods. pymel is designed to be safe to import into the root namespace so that scripts can be written much more concisely. All the maya.cmds methods are in pymel, except the conflicting ones ( file, filter, eval, help, and quit) which can be found in the pymel.cmds namespace. All of the file command's functionality can still be found directly within the pymel namespace where the file command has been broken down into multiple functions -- one for each primary flag -- each inheriting all of the same sub-flags and default values.

instead of this: 
>>> cmds.file( exportAll=1, preserveReferences=1 )
you can do this: 
>>> exportAll( preserveReferences=1)

Class Hiearchy

the pymel module organizes the myriad mel commands into a a hierarchy of classes, and provides many additional helper methods. This design allows you to write much more concise and readable python code, surpassing even mel in many cases.
  • str
    • _BaseObj
      • Attribute
      • Node
        • Dag
          • Transform
          • Poly
          • Nurbs
          • Curve
the _BaseObj is a subclass of python's built-in string, which allows it to be easily printed, passed to unwrapped commands and used as a key in dictionaries. however, since it is immutable, when calling commands like 'rename', the calling instance will point to an invalid object after the rename, so you need to assign the result of the command to a new instance. also, attributes that refer to the renamed objects will become invalid. for example: 
>>> orig = Poly('pSphere1')
>>> new = orig.rename('crazySphere')
>>> print orig.exists(), new.exists()
(0, 1) #

Objects

in order to use the object-oriented design of pymel, you must ensure that the objects that you are working with are instances of pymel classes. to make this easier, this module contains wrapped version of the more common commands for getting lists of objects: ls, listRelatives, and various creation commands such as polySphere, createNode, among many others. these methods attempt to cast their results to the appropriate class type (check the code below under the headings 'Commands for Returning pymel objects' and 'Commands for Creating pymel objects'). maya's versions of these commands return unicode strings, which can be used to initialize a pymel class. for example:

you can convert from a string to a pymel class --- 
>>> s = cmds.polySphere()[0]
>>> type(s)
<type 'unicode'> # 
>>>     
>>> s = Poly(s) 
>>> type(s)
<class 'pymel.Poly'> #
or you can just use the pymel version of the command 
>>> t = polySphere()[0]
>>> type(t)
<class 'pymel.Poly'> #

to make life even simpler, there is a 'scene' keyword which allows you to quickly wrap your objects into pymel types.

for instance, instead of this: 
>>> Transform('pSphere2').ty.get()
you can do this: 
>>> scene.pSphere2.ty.get()

Setup

if you have not done so already, it is a good idea to setup a directory for your published python scripts so that they will be accessible from within maya. you do this by setting the PYTHONPATH environment variable in Maya.env. if you want to use your current maya scripts directory simply add this to your Maya.env:

PYTHONPATH = $MAYA_SCRIPT_PATH

then place the pymel directory in your PYTHONPATH directory.

next, to avoid having to import pymel every time you startup, if you have not done so already, you can create a userSetup.mel file, place it in your scipts directory and add this line:

python("from pymel import *");

Note, that if you are running a linux/unix variant, including osx, and you have your PYTHONPATH set in a shell resource file, this value will override your Maya.env value if you launch maya from the terminal.soyou
